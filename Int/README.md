## 大整数类核心技术细节

#### 大整数类的核心设计目标，并不只是“支持更大的数”，而是：

- 在内部表示上保持统一、合法
- 在算术实现上分层处理复杂度
- 在性能上同时兼顾算法选择与内存行为
- 在工程实现上兼顾正确性、可维护性与可移植性

**以下6点将从内部设计、技术价值、底层实现与核心思想四个维度，总结大整数类的关键技术细节。**

1. **符号与绝对值分离（解耦思想）**
    - **设计** 
      - 使用单独的符号位`is_neg`表示正负
      - 数值部分仅存储无符号的绝对值 
    - **作用**
      - 避免`-0`等非法状态
      - 加减等操作拆成“绝对值运算 + 符号决策”
      - 比较大小时，可先比较符号，再比较绝对值
      - 降低有符号运算的各种分支复杂度，使实现更清晰
    - **实现要点**
      - 0值必须始终保持非负，确保0值只有唯一表示
      - 对于绝对值的操作推荐单独封装，避免与符号逻辑混杂，如`compareAbs()`、`addAbs()`，详见第5点
    - **核心思想：把复杂问题拆成两个简单问题：符号 + 绝对值**
  
2. **统一归一化函数`normalize()`（规范表示、边界处理的核心）**
    - **设计**
      - 在运算完成后统一调用`normalize()`
      - 清除高位无效的前导0
      - 强制0为非负，确保0值只有唯一表示
      - 保证内部表示始终合法、规范
    - **作用**
      - 各运算过程只需产生临时结果，边界处理统一在末尾调用`normalize()`处理
      - 减少重复的边界判断与清理逻辑
    - **实现要点**
      - `normalize()`的职责应稳定且单一：只负责“表示规范化”，不应混入额外逻辑
      - 0值的内部表示必须全局一致
    -  **核心思想：通过统一收口来维护对象的不变式**
  
3. **使用64位无符号整数`uint64_t`或32位`uint32_t`做存储单元（limb）**
    - **设计**
      - 使用固定宽度的无符号整数作为底层存储单元
      - 常见的选择是`uint32_t`或`uint64_t`
      - 每个存储单元保存大整数的一部分数值，整体通过数组/向量拼接表示完整大整数
    - **作用**
      - 相比按十进制逐位存储，固定宽度整数更CPU友好，运算效率更高
      - 存储单元越宽，通常limb数量越少，可降低遍历与管理开销
      - 无符号整数更适合处理进位、借位和位运算
    - **实现要点**
      - `uint64_t`并不总是优于`uint32_t`，具体选择取决于编译器以及中间结果的处理方式
      - 若使用`uint64_t`，乘法和加法的中间结果需要更宽类型存储或者额外拆分处理，详见第6点
    - **核心思想：底层设计应尽量硬件友好，同时兼顾可实现性与可移植性**

4. **低位在前存储（为运算流程服务）**
    - **设计**
      - 将大整数按多个存储单元（limb）拆分后，采用低位在前，高位在后的顺序存储
      - 该布局仅用于内部数据结构的组织，不影响外部输入输出的正常顺序
    - **作用**
      - 加法、减法、乘法等算术通常都从低位开始处理，低位在前可正向遍历
      - 进位和借位向更高位传播，与人类逻辑相一致
      - 避免配合位运算而频繁做额外的下标换算
    - **实现要点**
      - 输出十进制字符串时，可在转换阶段单独处理显示顺序，不必让内部结构迁就外部表现
    - **核心思想：存储结构应优先服务于计算，而不是表面形式**

5. **拆分绝对值层操作（分层设计）**
    - **设计**
      - 将不涉及符号的基础操作独立封装为绝对值层函数
      - 常见形式包括：
        - `compareAbs()`：比较两个绝对值大小
        - `addAbs()`：计算两个绝对值之和
        - `aubAbs()`：计算两个绝对值之差（大绝对值减小绝对值）
      - 对外的有符号加减乘除操作先处理符号，再调用这些基础函数
    - **作用**
      - 避免每个运算函数内同时处理“数值逻辑”与“符号逻辑”
      - 减少实现中的条件分支数量
      - 提高代码的可复用性和可测试性，~~其实是本人经常写出bug，方便debug~~
    - **实现要点**
      - 绝对值层函数只关注无符号数本身，不应掺杂符号处理的逻辑
      - `subAbs()`一般有明确的前置条件，如被减数的绝对值不小于减数
      - 有符号运算函数负责判断调用哪一个绝对值层函数，并在最终结果上恢复正确符号
    - **核心思想：通过分层把复杂逻辑拆开，让每一层只处理一种职责**

6. **为中间结果预留更大范围（防止溢出）**
    - **设计**
      - 在单个limb的加减乘中，中间结果可能超出当前存储单元的表示范围
      - 因此在实现时使用更宽的中间表示，或显示拆分高低位结果
      - 例如：
        - 加法时保存`a + b + carry`
        - 乘法时保存`a * b + 累加值 + carry`
    - **作用**
      - 避免中间结果溢出导致错误的进位、借位或乘法结果
      - 让底层运算更安全稳定
      - 为高位截取、低位保留、进位转播提供可靠基础
    - **实现要点**
      - 当limb使用`uint64_t`时，中间结果需要更宽的类型、编译器扩展或手动高低位拆分
      - 当平台不能提供更宽的整数类型时，需要使用分解乘法、手写进位处理等solution
    - **核心思想：底层计算不能只看最终存储范围，必须保证中间计算过程同样安全**
